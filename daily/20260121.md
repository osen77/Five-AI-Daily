# 前言

对于各大AI实验室而言，今年是消费者产品发布的爆发之年。OpenAI推出了数十项AI功能——包括GPT-4o图像生成、独立的Sora应用以及群聊功能。Gemini发布了几款极具病毒传播力的图像和视频生成模型（Nano Banana和Veo）。其他实验室如Anthropic、Perplexity、xAI和Meta也都推出了面向消费者的新工具，涵盖聊天、编程、搜索、图像生成等领域。

结果如何？可以说是喜忧参半。

**总体而言，AI的使用率在上升。但在通用助手方面，绝大多数消费者仍然只使用一款产品。在今年大部分时间里，ChatGPT的周活跃用户中，访问过其他大模型提供商的比例不到10%。**来自Yipit的数据也显示消费支出同样高度集中——只有9%的消费者同时订阅了ChatGPT、Gemini、Claude和Cursor中的一项以上。虽然大语言模型助手的竞争可能不是“赢家通吃”，但目前的局面看起来更像是“赢家拿走大头”。

**继2023年成为史上最快达到1亿周活的产品后，ChatGPT保持了巨大的消费者采用率，估计跨平台周活已达8亿至9亿。Gemini在网页端的规模是ChatGPT的34%，在移动端为40%。在用户参与度和留存率方面，ChatGPT同样占据主导地位。**根据Yipit的数据，ChatGPT的日活/月活比率为36%，几乎是Gemini（21%）的两倍。同样，第12个月的桌面端用户留存率为50%，也是Gemini（25%）的两倍。

然而，最近几个月Google开始火力全开，Gemini也看到了成效。据Yipit数据，ChatGPT桌面用户的同比增长率为23%，而Gemini高达155%。而且，Gemini的增长率在过去五个月中实际上还在加速，即便是在更大的用户基数上——像Nano Banana这样的新图像模型是主要驱动力。

Gemini的另一个亮点是什么？付费用户。Gemini Pro的订阅量同比增长近300%，而ChatGPT为155%。此外，虽然ChatGPT的免费用户留存率远好于Gemini，但付费用户的留存率却在同一个水平线上——以最近的群组测算，第12个月时ChatGPT为68%，Gemini为57%。

抛开数据不谈——今年谁赢得了消费者的心智份额？我们在此拆解了本年度最大的进展，以及我们对模型公司2026年发展方向的预测。

# OpenAI

ChatGPT以绝对的霸主地位进入了这一年。他们在3月推出了今年第一个病毒式的大型发布——4o图像生成功能，还记得那次大规模的吉卜力风格热潮吗？高峰期每小时新增100万用户。OpenAI今年的消费者战略是通过现有的ChatGPT界面推送新模型和新体验。

举几个例子？每日更新、群聊、录音转写、购物研究、任务和学习模式都是新体验，而Agent和个性化则是对常规ChatGPT用法的增强。我们认为，这些新体验中没有一个在通过使用率或留存率方面真正实现“突围”。在现有ChatGPT界面的限制下，很难提供一流的产品体验。

ChatGPT也在加倍努力成为个人管理工作的中心。**今年推出了“连接器”，你可以让ChatGPT访问各种工作应用（如G Suite、Microsoft、Notion、Stripe、Slack），从而提取上下文并执行操作。**而ChatGPT自家的Agent理论上可以帮助生成演示文稿和分析报告，但在部分测试场景下，仍存在响应速度偏慢、功能稳定性待优化的情况。

在新界面方面，这里有两个显著的例外。**首先是Sora，它在今年推出了独立的应用程序。作为一款创意工具，它非常成功，全球下载量超过1200万。但作为一款社交应用，它就不那么成功了，SensorTower估计其30天留存率低于8%。**相比之下，顶级消费者应用的D30留存率通常在30%以上。

**ChatGPT的第二个产品扩展是Atlas，即该公司的浏览器。ChatGPT在这方面入局稍晚，继Perplexity的Comet和The Browser Company的Dia之后才推出Atlas——但这是一款功能强大且执行力出色的产品。**Atlas目前仍仅限于Mac平台，Yipit估计访问过Atlas下载页面的ChatGPT用户不足5%，因此使用量可能仍然相当小。

**明年，我们将关注他们如何推进Apps——这是该公司允许第三方开发者在ChatGPT中构建体验的新基础设施。**如果成功，ChatGPT有机会成为十多年来第一个真正的新消费者平台。但是，在提供足够的空间以支持神奇的消费者体验方面，他们将面临与“连接器”同样的阻力。

# Google

谷歌自Bard时代以来已经取得了长足的进步。该公司在2025年无论是新模型还是新产品方面都迎来了爆发式增长。与OpenAI不同，谷歌为新产品创造了更多的“接触面”——该公司不仅在Gemini和其他接入点（如AI Studio,Labs,Flow）中发布模型，还会打造独立的体验。

**在模型方面，谷歌凭借Nano Banana和Nano Banana Pro赢得了消费者的时代精神。**据谷歌副总裁Josh Woodward称，**在发布的第一周，Nano Banana就生成了2亿张图像，并为Gemini带来了1000万新用户。**虽然在纯粹的使用量上略逊一筹，但谷歌的Veo 3视频模型可以说是AI视频领域的突破性时刻，因为它结合了视觉和音频生成。

**在产品方面，NotebookLM可能是谷歌成功推出新界面的最佳案例。**NotebookLM最初在2024年9月走红——且使用量至今仍在增长。截至11月，其网页端用户同比翻了一番以上。其移动应用于5月推出，目前月活跃用户已达800万。该产品也在不断进化，更新了幻灯片生成、视频概述和图表等功能。

谷歌今年还发布了许多其他产品，但反响相对平淡——如Portraits、Doppl、Whisk、Gems等等。**但是，与OpenAI将所有东西都“塞进”ChatGPT的策略相反，这些发布并没有让核心的Gemini体验变得杂乱无章。**它们可以独自沉浮，就像NotebookLM成功了一样！这里最显著的改进空间实际上是**入口**——搞清楚在哪里访问新产品以及需要什么样的账户，目前还很令人困惑。

谷歌在将AI嵌入其核心产品方面取得了一些进展，尽管这方面的速度较慢。**Gemini现在可以在Chrome浏览器中使用，直接与网页进行交互。其他重要的发布包括Gmail中的“帮我写”和“帮我安排”，以及GoogleMeet中的“帮我记笔记”和实时语音翻译。现有的Google Suite拥有巨大的分发优势，尽管感觉上这些产品的“AI原生”版本可能会带来更强大的体验。**

**谷歌在模型方面拥有一些高光时刻，Gemini正在积蓄力量，而NotebookLM在主流消费者AI使用方面继续成为一个极具亮点的产品**。话虽如此，没有让他们的产品更易于访问确实是一个真正的错失良机。

明年，我们将关注他们是否会将AI带入更多的核心体验中，因为ChatGPT正在持续侵蚀搜索业务。AI模式于5月在Google Search中首次亮相，但据Yipit数据，只有约2%的周活跃用户在与之交互。谷歌最近演示了Disco，它能将你的搜索标签页重组为交互式的、定制化的Web应用——这感觉像是我们见过的谷歌在创新核心UI方面做出的“最大胆的尝试”。

如果不想遗漏今年在消费者AI领域有所作为的其他核心玩家，我们必须提及以下几家：

# Anthropic

Anthropic仍然非常专注于“专业消费者”用户——尤其是在此之内的技术型用户。他们几乎所有的消费者端发布都围绕着Claude或扩展Claude，如Claude for Chrome,Claude Desktop。

他们推出了一些非常强大的工作新原语——Skills和Artifacts是最值得注意的，这些功能感觉是专为更成熟的用户设计的。**Anthropic也是大实验室中第一家致力由AI生成幻灯片、文档和模型的公司——在我们的测试中，其产品的速度和可靠性远超ChatGPT的Agent。**

**Claude终于在一些核心功能上赶上了ChatGPT，包括语音模式、记忆、网络搜索和研究，即Deep Research版本。然而，这感觉更像是重建“基本盘”，而不是试图吸引新的受众或用户。**

明年，我们预计Anthropic将继续投资Claude Code，该产品在六个月内就达到了10美元的年化营收。我们预计会看到其他针对同类用户的产品发布——也许是一流的数据分析界面，或其他类似的产品。

# Perplexity

**Perplexity今年同样加倍押注于专业消费者用户，但重点放在技术门槛较低的“效率发烧友”身上。他们最大的发布是Comet（可以说这是第一个真正的AI浏览器）和邮件助手。**虽然这两款产品都没有在大众层面爆火，但Comet拥有超过100万用户，并且仍然是我们最喜欢的AI浏览器。

Perplexity在购物领域也持续创新，继去年底首次推出AI驱动的购物助手后，今年他们推出了一个新的对话式商务智能体和一个能将照片转化为数字头像的虚拟试穿 工具。

该公司在今年3月宣布年化营收达到1亿美元，付费使用量同比增长6倍，月活跃用户超过2000万。Perplexity的收购动作可能暗示了他们明年的资源投入方向——他们收购了Read.cv（LinkedIn的竞争对手）、Visual Electric（专业消费者创意工具）、Invisible（智能体基础设施）和Carbon（外部数据源->LLM连接器）。

# xAI

Grok今年取得了长足的进步。**1月份，xAI推出了Grok独立应用——在此之前，它只是X平台上的一个聊天机器人。7月，xAI凭借伴侣功能的发布掀起了波澜（最初是Ani和Rudi）。**在那之前，大多数AI伴侣还只是语音——Grok通过全动画的角色和更具争议性的个性改变了游戏规则。

8月，xAI在Grok Imagine中推出了自己的图像和视频模型。**他们定期发布具有新功能的更新模型——Grok现在拥有带声音的文生视频和图生视频模型，包括带口型同步的语音。在能力提升方面，这可能是我们见过的最快爬坡速度。**

**根据SensorTower数据，Grok从今年年初的零用户增长到12月中旬的950万日活和3800万月活。**Grok也更深入地整合到了X应用中——你现在可以“长按”时间线上的任何图片来编辑它或用Grok制作视频。

我们预计明年会有更多动作，特别是在长格式或更具互动性的内容方面。Elon公开宣布，要在2026年底之前看到Grok制作的“可观看的”电影和“很棒的”AI生成游戏。我们可能会看到Grok在X应用中扮演更重要的角色——Elon曾表示，算法将由模型控制，通过阅读和观看内容来为用户“匹配”他们应该感兴趣的帖子。

# Meta

对Meta来说，今年是较为坎坷的一年。**该公司于4月推出了一款名为MetaAI的独立应用——旨在作为与你的AI助手聊天的场所。但该应用在6月陷入了病毒式传播的负面舆情，因为人们发现老用户在不经意间将他们与聊天机器人的对话分享到了主Discover信息流上。**该应用在9月重新发布，并在Meta与Midjourney达成合作后，推出了“Vibes”功能——一个AI生成的短视频滚动信息流。

这波“Vibes”（ZP注：双关，意指氛围）并未达到惊艳的效果——但Meta AI应用显示出了持续的增长，主要是在美国以外地区。据SensorTower数据，截至12月中旬，其日活达到420万，月活达到2660万。

Meta今年因挖角了一支全明星研究团队而闻名，他们在现有的应用程序（Facebook,Instagram,和WhatsApp）中拥有巨大的分发优势。我们开始看到他们将模型整合到这些产品中——比如Instagram Reels中的AI翻译——我们期待看到明年会有什么新动作。

# 创业公司的机会在哪里？

尽管模型巨头们动作频频，但我们从未对创业公司在消费者AI领域能构建的东西感到如此兴奋。**现在很清楚大实验室的注意力放在哪里** **：**

**(1)模型本身——这只会利好应用层公司；(2)主要通过其现有产品交付的新功能和界面。**

这为致力于构建专用消费者体验的创始人留下了巨大的空白机会。**模型公司没有直觉，或者坦率地说，没有精力和资源去在他们的核心能力领域之外进行创新。消费者领域的创始人甚至可能从ChatGPT即将推出的第三方应用发现和分发机制中受益。**

我们已经在这里看到了许多例子——那些“突围”并获得数百万消费者用户，且收入增长比以往任何时候都快的消费者AI产品包括Replit、Gamma、Character AI、Suno、Eleven Labs、Manus、Krea、Lovable。这些产品都拥有观点鲜明且专注的界面，赋予了消费者超越模型公司自身所能提供的“超能力”。

# [五条0121] Skywork Design Agent：专为非设计师打造的AI利器
发布日期：2026/01/18

![](https://mmbiz.qpic.cn/sz_mmbiz_png/xciaZIibdKg1Vc5cia2IzibGEq4xb5QNH3BhBIDtfMzerY20rlcEha1oOu2LAtRNCGZlkiagRUMic0YBdDdJOKP0tRew/640?wx_fmt=png&from=appmsg&tp=webp&wxfrom=5&wx_lazy=1)

我们发现，在AI 设计工具爆发的当下，非设计专业背景的打工人却经常陷入 “**工具越智能，使用越迷茫**” 的困境。差生文具多，但由于这些AI工具有着如操作门槛高、效果不可控等痛点，让多数 Design Agent 沦为 “演示神器”，很难真正融入日常办公场景。尤其对于运营、教育、创业等领域的非设计人士，他们往往需要提前做攻略且反复尝试，但设计出来的视觉作品仍然难以直接交付，严重制约了视觉内容创作的效率与质量。

# [五条0121] SubAgent vs. Skills：构建更强AI的两种路径
发布日期：2026/01/19

![](https://baoyu.io/blog/2026/01/19/skills-vs-subagent/imgs/cover.png)

一句话解释 SubAgent 和 Skills 的区别：**AI Agent 就像操作系统，Skills 就像应用程序，SubAgent 也是操作系统，不过预装了应用，并且像虚拟机一样运行，独立上下文。**

## Skills：教 Agent 新技能

**Skills 是一份“操作手册”。**

你招了个聪明的助理，但他对你的业务一无所知。每次布置任务，你都得从头解释流程、注意事项、工具用法。Skills 就是把这些知识写成文档，Agent 需要时自己去翻阅，不用你反复交代。

比如我写了一个“给文章配图”的 Skill，它告诉 Agent：

* 怎么分析文章找出需要配图的位置
* 有哪些风格可选（科技感、温暖、极简……）
* 怎么写图像生成的提示词
* 图片存哪里、怎么命名

Agent 启动时只记住“有这么个配图技能”，大概 100 个 token，真正要用时才去读详细内容。这叫\*\*“渐进式加载”\*\*，好处是装一堆 Skill 也不会撑爆上下文。

![](https://s.baoyu.io/imgs/2026-01-19/skills-vs-subagent/illustration-skills-architecture.png)

Skills 按需加载示意图

## SubAgent：开个虚拟机干活

**SubAgent 是独立运行的“专家助手”。**

它有自己的上下文窗口，相当于自己的“工作记忆”。干完活只把结果交回来，中间过程不会污染主 Agent 的思维空间。

你是项目经理，手下有三个专家。你不需要盯着他们每一步操作，只需要告诉他们任务目标，等他们交报告就行。

![](https://s.baoyu.io/imgs/2026-01-19/skills-vs-subagent/illustration-subagent-context.png)

SubAgent 独立上下文示意图

## 一个真实案例：给文章配图

假设我写完一篇文章，想配三套不同风格的插图方案，让用户自己选。

**如果只用 Skills：**

主 Agent 分析文章 → 设计三种方案 → 调用画图 Skill 画第一套 → 画完再画第二套 → 再画第三套

问题来了：

1. **上下文爆炸**：三套图的生成过程全堆在主 Agent 的“脑子”里，几十张图的提示词、中间结果、错误重试……全占着位置
2. **速度慢**：必须一个接一个画，串行执行

**如果用 SubAgent + Skills：**

主 Agent 分析文章 → 设计三种方案 → 同时启动三个 SubAgent

每个 SubAgent：

* 预装了“画图 Skill”
* 拿到文章路径和自己负责的方案
* 独立去读文章、生成提示词、调用画图工具、插入图片
* 完成后只返回：新文章的 URL + 简短摘要

主 Agent 这边呢？**上下文干干净净**，只有三个结果链接。用户想看哪套，再去读取对应的 URL。

![](https://s.baoyu.io/imgs/2026-01-19/skills-vs-subagent/illustration-workflow-comparison.png)

纯 Skills vs SubAgent+Skills 流程对比

## 什么时候用哪个？

**用 Skills：**

* 任务简单，主 Agent 全程掌控
* 需要的知识可以复用
* 想节省上下文，按需加载

**用 SubAgent：**

* 子任务复杂、耗时长、中间过程繁琐
* 需要并行处理多个独立任务
* 想保持主 Agent 的“思维清晰”

## 一句话总结

Skills 是应用程序，装在主系统里按需调用；SubAgent 是虚拟机，独立运行完再把结果交回来。**任务简单用应用，任务复杂开虚拟机。**
